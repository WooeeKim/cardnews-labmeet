# 다음 할 일

비슷한 카드뉴스·캐러셀 생성 도구들을 참고해서, 우리 스킬에 더할 점을 정리했습니다.

> 참고: 조사 당시 환경에서 웹 접근이 막혀 있어, 아래 5개 도구는 일반적으로 알려진 구조를 기준으로 정리했습니다. 정확한 현재 URL이나 기능은 쓰기 전에 한 번 확인하세요. 배울 점과 할 일 자체는 그대로 유효합니다.

## 참고한 도구

1. **Carbon / ray.so** (carbon.now.sh, ray.so). 텍스트·코드를 카드 이미지로 export. 모든 설정을 URL 파라미터로 직렬화해서 같은 카드를 다시 만들기 쉽고, export 시 2~3배 해상도로 굽는다.
2. **html-to-image / Playwright** (github.com/bubkoo/html-to-image). HTML/DOM을 픽셀 그대로 PNG로 굽는다. 슬라이드 div 하나씩 정확히 1080×1350으로 캡처할 수 있다.
3. **Satori** (github.com/vercel/satori). HTML+CSS를 브라우저 없이 SVG로 변환. 폰트를 명시적으로 임베드해서 환경이 달라도 똑같이 렌더되고, 지원 CSS를 제한해 레이아웃이 안 깨진다.
4. **미리캔버스 / 망고보드** (miricanvas.com, mangoboard.net). 국내 카드뉴스 표준. 표지·본문·엔딩이 한 세트로 묶여 톤이 자동으로 통일되고, 페이지 번호와 "넘기세요" 화살표가 기본이다. 한 장에 한 메시지, 큰 타이포 후킹.
5. **Marp / Slidev** (marp.app, sli.dev). 마크다운을 슬라이드로 변환. 콘텐츠와 테마를 완전히 분리하고, 슬라이드별로 전역 규칙을 덮어쓸 수 있다. CLI 한 줄로 PNG·PDF 일괄 export.

## 배울 점

우리가 이미 잘 하는 것: 역할별 슬라이드 분리(cover/interview/closing), 전역 규칙 + 슬라이드별 오버라이드. 이건 위 도구들도 공통으로 쓰는 좋은 구조라 그대로 유지한다.

아직 약한 네 군데: 이미지 export가 없다, 폰트를 CDN에 의존한다, 테마가 코드에 흩어져 있다, 재현·버전관리 장치가 없다.
